## Notes

The frontend uses prototype data to demonstrate AI recommendation, academic risk prediction, study resource filtering, analytics dashboard features, user management, course management, academic task tracking, home dashboard summaries, and generated study schedule features.

Abdulrahman's part focuses on the User Management and Academic Tracking subsystems. The implemented pages allow students to access the welcome page, log in, sign up, view their profile, manage courses, manage academic tasks, view the home dashboard, and view a generated study schedule. The course and task pages use LocalStorage to support adding, editing, deleting, filtering, and updating prototype data.

In the complete system, these frontend pages will be connected to the backend database so that user accounts, courses, tasks, schedules, academic progress, notifications, and AI-generated outputs are based on each student's actual data.
